# Security Policy

## Supported versions

The main branch is actively maintained. Please ensure you are testing against the latest commit or the most recent tagged release before reporting issues.

## Reporting a vulnerability

1. **Do not open public issues for security bugs.**
2. Email the maintainer (see repository contact info) with the following details:
   - A description of the vulnerability and potential impact.
   - Steps to reproduce or proof-of-concept code.
   - Any mitigating factors you identified.
3. We will acknowledge receipt within five business days and keep you informed as we triage and address the report.

## Disclosure process

- We aim to provide a fix within 30 days for high-impact issues.
- After a patch is released, we will credit reporters (if desired) in the release notes.

Thank you for helping keep this project safe for our community.
