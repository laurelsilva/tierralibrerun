# Athlete Experience Roadmap

## Objective

Make the athlete journey obvious, low-friction, and confidence-building from
application through race completion.

## Core Principles

1. Every stage must answer: "What happens next?"
2. Status without action is not enough; every status should map to a checklist.
3. Keep communication mirrored across email and in-app UI.
4. Reduce uncertainty around race timing and onboarding expectations.

## Phase 1 (High Impact, Low Risk)

1. Dashboard checklist (fund flow):
   - Confirm participation
   - Registration complete
   - Join Slack
   - Post intro
   - Mentor matched (if requested)
2. Add explicit "next step" messaging per workflow stage.
3. Add race countdown card and "race passed" notices.

## Phase 2 (Operational Quality)

1. In-app notification feed:
   - Stage changed
   - Action needed
   - Registration completed
   - Mentor assigned
2. Structured close reasons for clearer reporting and communication.
3. One-click support request from dashboard ("I need help with this step").

## Phase 3 (Delight + Retention)

1. Milestone timeline:
   - Application submitted
   - Approved
   - Confirmed
   - Registered
   - Onboarded
   - Race outcome
2. Post-race reflection module:
   - Collect lightweight feedback
   - Feed community stories/testimonials
3. Re-engagement pathways for no-show/drop outcomes:
   - Context-sensitive "ready to try again?" flow

## Product Metrics To Track

1. Time from `AWAITING_CONFIRMATION` to `CONFIRMED`
2. Time from `CONFIRMED` to `REGISTERED`
3. Onboarding completion rate before race day
4. `%` of applications ending in `NO_SHOW_OR_DROPPED`
5. Dashboard action completion rate per stage
